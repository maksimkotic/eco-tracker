# Структура проекта Eco Tracker

## Назначение
Eco Tracker — серверное веб-приложение на Node.js/Express для ведения экологичных привычек, отметок выполнения и достижений с ролевым доступом (user/moderator/admin).

## Корень проекта
- `app.js` — точка входа приложения: middleware, сессии, layout, маршруты, обработка ошибок.
- `package.json` — зависимости и npm-скрипты (`start`, `dev`, `init-db`, `reset-db`, `check`).
- `package-lock.json` — фиксированные версии пакетов.
- `STRUCTURE.md` — этот документ.

## Каталоги и ответственность

### `controllers/`
Бизнес-логика по HTTP-запросам.
- `authController.js` — регистрация, вход, выход.
- `habitController.js` — создание/редактирование/удаление привычек, отметки выполнения.
- `achievementController.js` — просмотр и логика достижений.
- `profileController.js` — профиль пользователя и персональные настройки.
- `moderatorController.js` — функции модератора для привычек/достижений.
- `adminController.js` — админ-панель: пользователи, роли, логи, системные операции.

### `routes/`
Маршруты Express, группировка по разделам приложения.
- `index.js` — главная и общие страницы.
- `authRoutes.js` — авторизация.
- `habitRoutes.js` — привычки.
- `achievementRoutes.js` — достижения.
- `profileRoutes.js` — профиль.
- `moderatorRoutes.js` — модераторские эндпоинты.
- `adminRoutes.js` — административные эндпоинты.

### `models/`
Sequelize-модели и связи.
- `User.js` — аккаунт, профиль, очки, активность.
- `Habit.js` — привычка, цель, частота, прогресс.
- `Checkin.js` — факт выполнения привычки.
- `Achievement.js` — достижение и условия выдачи.
- `UserAchievement.js` — связь пользователь ↔ достижение.
- `Role.js` — роль и набор разрешений.
- `index.js` — инициализация Sequelize и ассоциаций.

### `middlewares/`
Промежуточные обработчики доступа.
- `auth.js` — проверка сессии/пользователя.
- `role.js` — проверка роли.
- `permissions.js` — проверка разрешений.

### `views/`
EJS-шаблоны интерфейса.
- `layout.ejs` — общий layout.
- `index.ejs` — главная страница.
- `errors/` — страницы ошибок (`403`, `404`, `500`).
- `auth/` — вход/регистрация.
- `habits/` — страницы привычек (список, создание, редактирование, просмотр, отметка).
- `achievements/` — пользовательские достижения.
- `profile/` — профиль и редактирование.
- `moderator/` — интерфейс модератора.
- `admin/` — интерфейс администратора (dashboard, users, roles, logs).
- `static/` — статические информационные страницы (`contacts`, `privacy`, `terms`, `guide`).

### `database/`
Инициализация и наполнение базы.
- `init.js` — создание схемы/сидирование.
- `bootstrap.js` — вспомогательная логика начального развёртывания.

### `utils/`
Служебные функции.
- `logger.js` — прикладное логирование действий и событий.

## Поток запроса
1. Запрос приходит в `app.js`.
2. Проходит через middleware (сессия, auth, role/permissions, flash, security headers).
3. Попадает в соответствующий файл из `routes/`.
4. Обрабатывается методом контроллера из `controllers/`.
5. Контроллер работает с моделями из `models/`.
6. Рендерится EJS-шаблон из `views/` или возвращается JSON.

## Проверка проекта
- Базовая проверка синтаксиса: `npm run check`.
- Запуск приложения: `npm start`.
- Режим разработки: `npm run dev`.
